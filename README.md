# FloodSense 🌊

**Intelligent flood prediction and early warning system for Kakinada, Andhra Pradesh.**

FloodSense combines real-time weather data, machine learning, multilingual alerts, and SMS notifications to detect flood risk and warn communities in their native language.

---

## Features

- **Live weather monitoring** — fetches real-time rainfall and humidity from the Open-Meteo API
- **ML flood prediction** — Random Forest classifier trained on historical flood data
- **Simulate mode** — inject extreme flood conditions for demos and testing
- **Multilingual alerts** — translates warnings into Telugu, Hindi, Tamil, Kannada, Malayalam, and English using Facebook's NLLB-200 model
- **SMS via Twilio** — sends translated flood alerts directly to registered phone numbers
- **Auto-refresh dashboard** — live data updates every 60 seconds

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/your-username/floodsense.git
cd floodsense
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

> **Note:** The NLLB-200 translation model (~600 MB) will download automatically on first run.

### 3. Train the ML model

Run this once before starting the app:

```bash
python train_model.py
```

This generates `flood_model.pkl` in the project root.

### 4. Configure environment variables

Create a `.env` file or export these variables in your shell:

```bash
export TWILIO_ACCOUNT_SID="your_account_sid"
export TWILIO_AUTH_TOKEN="your_auth_token"
export TWILIO_FROM_NUMBER="+1xxxxxxxxxx"
export TWILIO_TO_NUMBER="+91xxxxxxxxxx"
```

> Without Twilio credentials, the app runs in **demo mode** — alerts are printed to the console but no SMS is sent.

### 5. Run the app

```bash
python app.py
```

### 6. Open in your browser

```
http://127.0.0.1:5000
```

## How It Works

1. **Data collection** — `data_fetch.py` polls the Open-Meteo API for current rainfall (mm) and relative humidity (%) for Kakinada (lat: 15.91, lon: 79.74). Water level is estimated as `rainfall × 0.2`.

2. **Flood prediction** — The trained Random Forest model takes `[rainfall, water_level, humidity]` as input and outputs a binary prediction: `0` (Safe) or `1` (Flood risk). A fallback rule-based scorer is used if the model file is missing.

3. **Alert translation** — When a flood is detected, `translate_llm.py` uses Facebook's `nllb-200-distilled-600M` model to translate the alert message from English into the selected regional language.

4. **SMS dispatch** — `send_sms.py` sends the translated message via the Twilio REST API.

---

## Model Details

The classifier is trained on synthetic flood data with the following features:

| Feature | Description | Range |
|---------|-------------|-------|
| `rainfall` | Rainfall in mm | 10 – 300 |
| `water_level` | Estimated water level in meters | 0.5 – 15 |
| `humidity` | Relative humidity in % | 55 – 95 |

Training uses a `RandomForestClassifier` with 100 estimators. Retrain at any time by running `train_model.py`.
